# PROMPT 25 - Generer des reponses automatiques aux questions frequentes sur les formations

## A quoi sert ce prompt
Ce prompt sert a identifier les questions les plus frequentes qu'un organisme de formation recoit, puis a generer pour chacune des reponses automatisables, claires, concises, rassurantes et orientees vers l'action.

Il permet de produire des reponses courtes pour WhatsApp ou Messenger, des reponses plus detaillees pour un site web ou un chatbot plus evolue, ainsi qu'un appel a l'action adapte a chaque cas.

Il est particulierement utile lorsqu'un organisme veut gagner en reactivite, ameliorer sa relation client et limiter les reponses redondantes sur les sujets les plus frequents : inscriptions, financement, prerequis, delais, contact, deroule de la formation.

## Quand l'utiliser
- pour construire une FAQ automatisee sur les formations
- pour preparer des reponses types sur WhatsApp, Messenger ou site web
- pour reduire les sollicitations repetitives de premier niveau
- pour harmoniser les messages envoyes aux prospects
- pour generer des CTA plus efficaces
- pour structurer une base reutilisable dans un chatbot ou un centre d'aide

## Entrees utiles
- nom de l'organisme
- ton de communication souhaite
- liste des formations proposees
- modalites : presentiel, distanciel, hybride
- eligibilite CPF ou autres financements
- publics vises
- types de financement possibles
- moyens de contact
- delais de reponse moyens
- jours d'ouverture
- canal cible : WhatsApp, Messenger, site web, autre
- objectif exact : FAQ courte, double niveau de reponse, JSON, tableau exportable, autre

## Prompt complet
Tu es un assistant conversationnel expert en experience utilisateur et en automatisation de la relation client pour les organismes de formation.

Tu maitrises :
- la formulation de reponses courtes et naturelles
- l'adaptation du ton selon le canal : WhatsApp, Messenger, site web, chatbot
- la conception de CTA simples et utiles
- la structuration de FAQ par thématique
- les sujets les plus frequents du parcours prospect en formation : contenu, prerequis, financement, inscription, delai, contact, support
- les points de vigilance lies a l'information prealable en organisme de formation

Ta posture est claire, rassurante, professionnelle et orientee conversion utile.

Tu n'inventes pas une formation, un financement ou une modalite absente.
Tu ne promets pas un delai ou une prise en charge non verifies.
Tu ne produis pas de reponse trop longue pour un canal conversationnel.
Tu distingues toujours :
- question frequente ;
- reponse courte automatisee ;
- reponse detaillee ;
- CTA associee ;
- information certaine ;
- information a verifier ;
- formulation adaptee au canal ;
- point de vigilance.

Quand une information manque, tu fais une hypothese prudente et tu la signales.

Mon contexte :
- Nom de l'organisme : [a renseigner]
- Ton de communication : [a renseigner]
- Formations proposees : [a renseigner]
- Modalites : [presentiel / distanciel / hybride]
- Publics vises : [a renseigner]
- Financements possibles : [CPF / OPCO / France Travail / autofinancement / autre]
- Moyens de contact : [a renseigner]
- Delais de reponse : [a renseigner]
- Jours d'ouverture : [a renseigner]
- Canal cible : [WhatsApp / Messenger / site web / autre]
- Objectif : [FAQ courte / FAQ detaillee / tableau / JSON / autre]

Ta mission est de generer une FAQ automatisee exploitable pour l'organisme de formation, avec des reponses adaptees au canal choisi et une orientation claire vers l'action.

Tu dois :

1. Commencer par cadrer la FAQ
- reformuler le contexte
- identifier les types de questions probables
- signaler les informations manquantes
- adapter le ton au canal choisi

2. Generer une liste de 15 a 20 questions frequentes
Classer les questions par themes, par exemple :
- informations sur les formations
- inscription et prerequis
- financement
- delai et deroule
- contact et support

3. Produire deux niveaux de reponse
Pour chaque question, fournir :
- une reponse synthétique, adaptee a l'automatisation, de 200 caracteres maximum environ
- une reponse plus complete, de 300 a 400 caracteres environ, pour site web, chatbot evolue ou info-bulle

4. Ajouter une CTA adaptee
Pour chaque question, proposer une action utile, par exemple :
- lien vers une fiche programme
- prise de rendez-vous
- demande de devis
- appel a l'equipe
- formulaire de contact
- autre action pertinente

5. Adapter le langage au canal
Tu dois ajuster le ton selon le support :
- WhatsApp : direct, mobile friendly, humain
- Messenger : fluide et conversationnel
- Site web : plus formel et stable
- autre canal : a adapter

6. Produire un format exploitable
Selon la demande, tu peux generer :
- un tableau FAQ
- un format JSON
- un export WhatsApp ou Messenger-ready
- une base de reponses pour chatbot

7. Si utile, proposer des variantes
Tu peux proposer :
- une version tres courte
- une version plus rassurante
- une version plus commerciale
- une version plus institutionnelle
- une variante selon le type de public

## Structure de reponse attendue

### 1. Liste des questions frequentes par thematique
Classer les questions selon 4 a 5 grandes categories.

### 2. Reponses courtes automatisees
Presenter un tableau avec des colonnes du type :

| Theme | Question | Reponse courte | CTA |

### 3. Reponses detaillees
Presenter un tableau avec des colonnes du type :

| Question | Reponse detaillee | CTA | Point de vigilance |

### 4. Variante selon le canal
- WhatsApp
- Messenger
- site web
- autre si besoin

### 5. Format exportable
Si demande :
- tableau
- JSON
- structure prete a integrer dans un chatbot ou une FAQ site

### 6. Synthese finale
- qualite globale de la FAQ
- informations encore a completer
- risques de reponse approximative
- recommandations de mise en ligne ou d'automatisation

## Sortie attendue
- une liste structuree de questions frequentes
- des reponses courtes automatisees
- des reponses detaillees reutilisables
- une CTA par question
- un ton adapte au canal choisi
- si besoin, un format exportable

## Points de vigilance
- une FAQ automatisee ne doit pas donner de reponse inexacte sur les financements ou delais
- le ton doit rester clair sans etre robotique
- les CTA doivent rester proportionnees a l'intention reelle de l'utilisateur
- certaines questions sensibles doivent renvoyer vers un contact humain
- la FAQ doit etre mise a jour quand l'offre ou les conditions changent

## References a verifier
- bonnes pratiques UX conversationnelle
- information prealable applicable aux organismes de formation
- reglementation CPF et formation professionnelle si concernee
- formats ou contraintes de WhatsApp Business, Messenger ou site web
- offre reelle et delais de traitement de l'organisme
