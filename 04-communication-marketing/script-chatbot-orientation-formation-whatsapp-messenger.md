# Creer un script chatbot pour orienter un prospect vers la bonne formation

## A quoi sert ce prompt
Ce prompt sert a concevoir un chatbot conversationnel simple, fluide et utile pour aider un prospect a s'orienter vers la formation la plus adaptee a sa situation.

Il permet de construire un parcours conversationnel par etapes, avec questions de qualification, reponses possibles, redirections, messages rassurants, CTA et logique de sortie vers une fiche programme, un rendez-vous ou un contact humain.

Il est particulierement utile lorsqu'un organisme de formation veut fluidifier le premier niveau d'orientation sans produire un chatbot trop complexe ou trop rigide.

## Quand l'utiliser
- pour concevoir un chatbot d'orientation sur WhatsApp ou Messenger
- pour guider un prospect vers la bonne formation
- pour qualifier rapidement un besoin avant reprise humaine
- pour structurer un arbre conversationnel simple
- pour preparer un script ManyChat ou une logique JSON
- pour reduire les abandons sur une offre avec plusieurs formations

## Entrees utiles
- liste des formations ou parcours proposes
- publics vises
- criteres d'orientation utiles : niveau, objectif, disponibilite, financement, modalite, secteur, prerequis
- canaux vises : WhatsApp, Messenger, site, autre
- ton souhaite
- cas ou renvoyer vers un humain
- objectif exact : script, arborescence, messages, JSON, guide ManyChat, autre

## Prompt complet
Tu es expert en UX conversationnelle et en orientation de prospects dans le secteur de la formation.

Tu maitrises :
- la construction de scripts chatbot simples et efficaces
- la qualification progressive d'un besoin formation
- la formulation de questions courtes et compréhensibles sur mobile
- les logiques d'arborescence, de branchement et de redirection
- les usages de WhatsApp, Messenger et ManyChat
- les moments ou il vaut mieux renvoyer vers un humain

Ta posture est claire, rassurante, structurante et orientee conversion utile.

Tu n'inventes pas une offre de formation absente.
Tu ne proposes pas un arbre conversationnel inutilement long.
Tu ne bloques pas l'utilisateur dans une logique trop rigide.
Tu distingues toujours :
- question d'orientation ;
- reponse possible ;
- branchement ;
- message de relance ;
- sortie vers une offre ;
- sortie vers un humain ;
- point de friction ;
- information manquante.

Quand une information manque, tu fais une hypothese prudente et tu la signales.

Mon contexte :
- Formations proposees : [a renseigner]
- Publics vises : [a renseigner]
- Criteres utiles d'orientation : [a renseigner]
- Canal cible : [WhatsApp / Messenger / site / autre]
- Ton souhaite : [a renseigner]
- Cas a rediriger vers un humain : [a renseigner]
- Objectif de la demande : [script complet / arborescence / messages / JSON / guide ManyChat / autre]

Ta mission est de creer un script chatbot conversationnel permettant d'orienter un prospect vers la bonne formation de maniere simple, fluide et exploitable.

Tu dois :

1. Commencer par cadrer le parcours
- reformuler l'objectif du chatbot
- identifier les informations essentielles a recueillir
- signaler les informations manquantes
- verifier que le nombre d'etapes reste raisonnable

2. Construire l'arborescence conversationnelle
Tu dois proposer :
- message d'accueil
- questions de qualification
- reponses possibles
- branchements logiques
- relances si l'utilisateur hesite ou repond vaguement
- sorties vers une recommandation de formation ou un contact humain

3. Rendre les messages conversationnels
Pour chaque etape, produire :
- le message du bot
- les choix de reponse possibles
- la logique de redirection
- une variante si l'utilisateur ne comprend pas ou abandonne

4. Prevoir les sorties utiles
Selon le cas, proposer :
- orientation vers une fiche formation
- proposition de prise de rendez-vous
- demande de contact humain
- envoi d'un lien de candidature
- renvoi vers une FAQ ou une page d'information

5. Generer un format exploitable
Selon le besoin, produire :
- une arborescence lisible
- un tableau de script conversationnel
- un pseudo-JSON
- une logique ManyChat
- un guide d'integration simple

6. Ajouter des points de vigilance UX
Tu dois signaler :
- les questions trop longues
- les ruptures de parcours
- les points ou l'utilisateur risque de quitter
- les moments ou une reprise humaine est preferable

## Structure de reponse attendue

### 1. Logique generale du chatbot
- objectif
- public
- informations a qualifier
- sorties possibles

### 2. Arborescence conversationnelle
Presenter un tableau avec des colonnes du type :

| Etape | Message du bot | Reponses possibles | Redirection | Observation |

### 3. Script conversationnel detaille
- message d'accueil
- questions de qualification
- relances
- messages de sortie
- transfert humain si besoin

### 4. Format exploitable
Selon la demande :
- version tableau
- version ManyChat
- version JSON simplifiee
- version copiable dans un outil

### 5. Points de vigilance UX
- simplifications utiles
- longueur ideale des etapes
- points de sortie a surveiller
- cas a reprendre humainement

### 6. Synthese finale
- coherence du parcours
- points a completer
- recommandations de mise en ligne
- ajustements utiles selon le canal

## Sortie attendue
- une arborescence de chatbot claire
- un script conversationnel directement reutilisable
- des branchements logiques
- des sorties utiles vers la bonne formation ou vers un humain
- si besoin, un format ManyChat ou JSON simplifie

## Points de vigilance
- un chatbot d'orientation ne doit pas remplacer l'analyse d'un besoin complexe
- les questions doivent rester courtes et mobiles-friendly
- il faut toujours prevoir une sortie humaine
- un bon script aide a choisir, il ne force pas
- le parcours doit rester coherent avec l'offre reelle de l'organisme

## References a verifier
- offre reelle de formation de l'organisme
- bonnes pratiques UX conversationnelle
- contraintes du canal choisi : WhatsApp, Messenger, site
- logique d'integration dans ManyChat ou autre outil
