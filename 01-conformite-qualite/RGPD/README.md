# Prompts RGPD pour organismes de formation

Ce dossier contient des prompts destinés à aider les organismes de formation, CFA et formateurs indépendants à structurer leur démarche RGPD.

Ces prompts peuvent aider à :
- auditer un tableau, un formulaire ou un dossier ;
- repérer des données excessives ;
- préparer une cartographie des traitements ;
- structurer un registre RGPD ;
- construire un plan d’action ;
- formaliser des procédures internes ;
- préparer des checklists d’archivage, de suppression ou de minimisation.

## Point de vigilance essentiel

Avant d’utiliser un prompt avec une IA, ne copiez jamais directement un fichier contenant des données personnelles réelles si l’outil utilisé n’est pas maîtrisé ou contractuellement encadré.

Cela concerne notamment :
- noms et prénoms ;
- adresses email ;
- numéros de téléphone ;
- dates de naissance ;
- adresses postales ;
- numéros de sécurité sociale ;
- RIB ;
- données de santé ;
- informations liées au handicap ou à la RQTH ;
- commentaires individualisés sur des apprenants, salariés ou formateurs ;
- identifiants de connexion ;
- toute donnée permettant d’identifier directement ou indirectement une personne.

## Bon réflexe avant usage d’une IA

Avant toute analyse par IA :

1. Supprimer les colonnes nominatives.
2. Remplacer les personnes par des identifiants fictifs si nécessaire.
3. Supprimer les données sensibles.
4. Agréger les données quand c’est possible.
5. Ne conserver que les colonnes utiles à l’analyse.
6. Vérifier que le fichier ne contient pas d’informations cachées ou de commentaires.
7. Utiliser de préférence des données fictives, anonymisées ou fortement simplifiées.

Exemple :

À ne pas envoyer :

| Nom | Prénom | Email | RQTH | Résultat | Commentaire |
|---|---|---|---|---|---|

À privilégier :

| ID fictif | Type de parcours | Statut | Résultat anonymisé | Point de vigilance |
|---|---|---|---|---|

## IA et RGPD : principe simple

Le problème n’est pas l’IA en elle-même.

Le problème est ce qu’on lui transmet.

Une IA peut être utile pour :
- structurer une analyse ;
- identifier des risques ;
- reformuler une procédure ;
- construire un plan d’action ;
- prioriser des corrections ;
- générer une trame de registre.

Mais elle ne doit pas recevoir de données personnelles réelles sans cadre juridique, technique et contractuel adapté.

## Ce que ces prompts ne remplacent pas

Ces prompts ne remplacent pas :
- l’analyse d’un DPO ;
- l’avis d’un juriste ou d’un avocat ;
- une analyse d’impact AIPD lorsque celle-ci est nécessaire ;
- une vérification des contrats de sous-traitance ;
- une analyse complète des transferts hors Union européenne ;
- une décision de conformité engageant officiellement l’organisme.

Ils servent à préparer, structurer et accélérer le travail.

## Usage recommandé

Pour un usage sécurisé :

- travailler d’abord sur des cas fictifs ;
- utiliser des tableaux anonymisés ;
- documenter les hypothèses ;
- faire valider les livrables sensibles ;
- distinguer ce qui relève d’une obligation légale, d’une recommandation CNIL ou d’une bonne pratique interne.

## Classement conseillé des prompts

### 1. Prompts utilisables en autonomie

- audit d’un tableau ou formulaire ;
- checklist de minimisation ;
- checklist d’archivage ;
- mini diagnostic RGPD ;
- préparation d’un plan d’action.

### 2. Prompts à utiliser avec prudence

- registre des traitements ;
- politique de confidentialité ;
- mentions d’information ;
- audit des sous-traitants ;
- analyse des outils numériques.

### 3. Prompts à réserver à un usage accompagné

- gestion d’un incident ou d’une violation de données ;
- données de santé ou handicap ;
- mineurs ;
- transfert hors Union européenne ;
- analyse d’impact AIPD ;
- clauses contractuelles complexes.

## Règle d’or

Ne jamais chercher à “faire passer” des données personnelles dans l’IA.

Préparer les données avant.

Puis utiliser l’IA pour analyser la structure, les risques, les catégories, les durées, les rôles et les actions à mener.
