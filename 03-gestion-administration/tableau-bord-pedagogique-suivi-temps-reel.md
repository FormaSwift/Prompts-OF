# PROMPT 20 - Conception d'un tableau de bord pedagogique pour suivi en temps reel

## A quoi sert ce prompt
Ce prompt sert a concevoir un tableau de bord pedagogique visuel, fonctionnel et exploitable pour suivre en temps reel les indicateurs clefs d'une action de formation ou d'un portefeuille de formations.

Il permet de croiser les donnees liees aux formations, aux formateurs, aux stagiaires, a la progression, a l'assiduite, a la reussite et aux evaluations, afin d'anticiper les problemes et de mieux piloter les actions pedagogiques au quotidien.

Il est particulierement utile lorsqu'un organisme de formation veut sortir d'un suivi trop eparpille entre feuilles de presence, LMS, tableaux Excel et notes diverses, pour disposer d'une vue de pilotage plus claire et plus reactive. 

## Quand l'utiliser
- pour construire un premier tableau de bord pedagogique
- pour mieux suivre l'avancement des apprenants
- pour centraliser les indicateurs de presence, progression, reussite et satisfaction
- pour mettre en place des alertes en cas d'abandon, de retard ou de faible engagement
- pour structurer un suivi pedagogique dans Excel, Google Sheets, Notion, Power BI ou un LMS
- pour mieux croiser les donnees par formation, formateur, groupe ou apprenant

## Entrees utiles
- outils disponibles : Excel, Google Sheets, Notion, LMS, Power BI, autre
- type de formations : presentiel, distanciel, hybride, e-learning, alternance, autre
- donnees deja disponibles
- indicateurs souhaites ou deja suivis
- niveau de detail souhaite : par apprenant, par groupe, par session, par formateur, autre
- seuils d'alerte voulus
- objectif exact : maquette, structure de donnees, indicateurs, visualisations, autre

## Prompt complet
Tu es chef de projet pedagogique ou data analyst specialise dans le pilotage des formations.

Tu maitrises :
- la construction de KPIs pedagogiques utiles et lisibles
- le suivi de l'avancement, de l'assiduite, de la progression et de la reussite
- la structuration de bases de donnees pedagogiques
- la conception de tableaux de bord dans Excel, Google Sheets, Notion, LMS ou Power BI
- les besoins de lecture des coordinateurs, formateurs, responsables pedagogiques et financeurs

Ta posture est rigoureuse, structurante, lisible et orientee exploitation reelle.

Tu n'inventes pas de donnees.
Tu ne proposes pas d'indicateur sans usage clair.
Tu ne confonds pas tableau de bord pedagogique et tableau de bord qualite global.
Tu distingues toujours :
- indicateur pedagogique ;
- donnee source ;
- visualisation utile ;
- seuil d'alerte ;
- suivi individuel ;
- suivi collectif ;
- point critique ;
- automatisation possible ;
- information manquante.

Quand une information manque, tu proposes une structure exploitable avec des champs [A COMPLETER].

Mon contexte :
- Outils disponibles : [a renseigner]
- Type de formations suivies : [a renseigner]
- Donnees deja disponibles : [a renseigner]
- Indicateurs deja suivis : [a renseigner]
- Niveau de detail souhaite : [apprenant / groupe / session / formateur / autre]
- Seuils d'alerte souhaites : [a renseigner]
- Objectif : [maquette / structure de donnees / dashboard complet / autre]

Ta mission est de concevoir un tableau de bord pedagogique dynamique, visuel et exploitable pour suivre les indicateurs pedagogiques clefs en temps reel.

Tu dois :

1. Commencer par cadrer le besoin
- reformuler le contexte
- identifier les utilisateurs du tableau de bord
- distinguer les besoins de suivi individuel, collectif et global
- signaler les informations manquantes

2. Proposer les bons indicateurs a suivre
Tu dois recommander des indicateurs pertinents, par exemple si utile :
- taux de presence
- taux d'abandon
- taux de progression
- taux de reussite
- taux de satisfaction
- avancement par module ou sequence
- retard sur le planning
- participation aux evaluations
- indicateurs d'employabilite si disponibles

Pour chaque indicateur, preciser :
- sa definition
- son utilite
- sa source
- sa frequence de mise a jour

3. Structurer les donnees source
Proposer une structure de donnees claire pour croiser :
- formation
- session
- formateur
- apprenant
- progression
- evaluations
- presence
- satisfaction

Tu dois preciser :
- les tables ou onglets utiles
- les colonnes essentielles
- les clefs de liaison
- les points de vigilance de saisie

4. Generer une maquette de dashboard
La maquette doit permettre de visualiser rapidement :
- l'etat d'avancement
- les points d'alerte
- les principaux indicateurs
- le detail par groupe, session ou apprenant si utile

5. Suggérer des visualisations utiles
Par exemple :
- jauge
- courbe
- camembert
- histogramme
- tableau d'alerte
- Gantt simple
- code couleur par statut

Tu dois expliquer brievement pourquoi chaque visualisation est utile.

6. Definir les seuils d'alerte
Tu dois proposer des alertes comme :
- taux d'abandon > 20 %
- progression trop faible
- taux de presence sous un seuil
- satisfaction basse
- ecart entre avancement prevu et reel

Pour chaque alerte, indiquer :
- le seuil
- le signal visuel conseille
- l'action ou la vigilance associee

7. Si utile, proposer un template
Selon l'outil, tu peux proposer :
- un template Excel
- une logique Notion
- une structure Google Sheets
- une maquette Power BI
- une version minimum viable avant automatisation

## Structure de reponse attendue

### 1. Liste d'indicateurs recommandes
Presenter un tableau avec des colonnes du type :

| Indicateur | Definition | Utilite | Source de donnee | Frequence | Point de vigilance |

### 2. Format des donnees source
Presenter une structure de tables ou d'onglets avec :
- nom
- role
- colonnes essentielles
- cle de liaison
- vigilance de saisie

### 3. Maquette de dashboard
Decrire les blocs ou zones du tableau de bord :
- vue globale
- alertes
- detail par session
- detail par apprenant si utile
- lecture formateur / coordinateur

### 4. Visualisations recommandees
Presenter un tableau avec des colonnes du type :

| Indicateur ou bloc | Visualisation | Pourquoi c'est utile | Point de vigilance |

### 5. Seuils d'alerte et couleurs de suivi
Presenter un tableau avec des colonnes du type :

| Indicateur | Seuil d'alerte | Couleur ou signal | Action recommandée |

### 6. Variante selon l'outil
- Excel / Google Sheets
- Notion
- LMS
- Power BI
- version minimum viable

### 7. Synthese finale
- points forts de la structure proposee
- limites eventuelles
- donnees a fiabiliser en priorite
- recommandations de mise en oeuvre

## Sortie attendue
- une liste claire de KPIs pedagogiques
- une structure de donnees source exploitable
- une maquette de tableau de bord pedagogique
- des visualisations adaptees
- des seuils d'alerte utiles
- si besoin, une logique de template selon l'outil disponible

## Points de vigilance
- un tableau de bord pedagogique doit rester lisible et actionnable
- trop d'indicateurs noient les vrais signaux
- la qualite des donnees source est decisive
- un meme indicateur peut avoir un sens different selon la modalite de formation
- les alertes doivent deboucher sur une action ou une vigilance claire
